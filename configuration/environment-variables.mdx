---
title: "Environment Variables"
description: "Environment variables that control Claude Code behavior"
---

Claude Code reads a number of environment variables at startup and throughout a session. These complement `settings.json` for cases where you need machine- or process-level control without modifying files.

<Note>
`ANTHROPIC_API_KEY` is required when authenticating via API key. Without it (and without an active OAuth session), Claude Code cannot make API calls.
</Note>

## Authentication

| Variable | Default | Description |
|----------|---------|-------------|
| `ANTHROPIC_API_KEY` | ‚Äî | Anthropic API key used for direct authentication |
| `ANTHROPIC_BASE_URL` | ‚Äî | Override the Anthropic API base URL |
| `ANTHROPIC_UNIX_SOCKET` | ‚Äî | Connect to the Anthropic API over a Unix domain socket |
| `CLAUDE_CODE_OAUTH_TOKEN` | ‚Äî | OAuth bearer token injected by the calling environment |
| `AWS_BEARER_TOKEN_BEDROCK` | ‚Äî | Bearer token for AWS Bedrock authentication |

## Operational modes

| Variable | Default | Description |
|----------|---------|-------------|
| `CLAUDE_CODE_SIMPLE` | ‚Äî | Enable bare/simple mode: skips hooks, LSP, plugin sync, skill directory walk, attribution, background prefetches, and all keychain reads. Auth is strictly `ANTHROPIC_API_KEY` or `apiKeyHelper`. Equivalent to the `--bare` CLI flag |
| `CLAUDE_CODE_REMOTE` | ‚Äî | Enable remote container mode; sets `max-old-space-size=8192` and enables all hook events |
| `CLAUDE_CODE_REMOTE_SESSION_ID` | ‚Äî | Session ID for the current remote session |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` | ‚Äî | Disable non-essential network traffic (analytics, telemetry, etc.) |
| `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS` | ‚Äî | Disable experimental beta API features |

| Variable | Default | Description |
|----------|---------|-------------|
| `CLAUDE_CODE_DISABLE_AUTO_MEMORY` | ‚Äî | Disable the auto-memory system entirely. Set to `1` or `true` to turn off; `0` or `false` to force-enable even when `autoMemoryEnabled: false` is in settings |

## How environment variables interact with settings

Environment variables are evaluated at the process level and take effect for the lifetime of the Claude Code process. They are not written to `settings.json`.

The `env` field in `settings.json` lets you set environment variables that are applied to each Claude Code session:

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://my-proxy.example.com",
    "NODE_EXTRA_CA_CERTS": "/etc/ssl/my-ca-bundle.crt"
  }
}
```

<Note>
Truthy values for boolean environment variables are `1`, `true`, `yes`, and `on` (case-insensitive). Falsy values are `0`, `false`, `no`, and `off`. An unset variable is treated as falsy.
</Note>
